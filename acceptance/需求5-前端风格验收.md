## 改动范围

- 建立 “Library Atlas / 馆藏运营工作台” 统一视觉语言：深墨蓝导航、青绿业务强调、暖灰工作区、低饱和状态色。

- 保持既有请求路径、请求字段和 API 数据结构不变；未修改后端、数据库、`request.js`、`application.properties`、`Result.java` 或路由 history。
- 生产构建后的最终 `vue/dist` 已保留，并通过 `Dockerfile.frontend` 构建镜像。

## 响应式与浏览器检查

Playwright 分别以 1280×800 和 1024×768 登录管理员账号并访问 Dashboard、Book、BookWithUser、LendRecord、User。

- 两档视口下五个主页面的 `body.scrollWidth` / `documentElement.scrollWidth` 均等于视口宽度，无页面级无意义横向滚动。
- 1280px 下侧栏保持完整导航；1024px 下内容区收紧，指标卡由四列调整为两列。
- 数据表格使用独立 `.table-scroll`，宽表只在表格区域产生可理解的横向滚动，固定操作列仍可见。
- 两档视口均无浏览器 console error 或 page error。
- 登录及主要页面请求均实测返回 HTTP 200：`/api/user/login`、`/api/dashboard`、`/api/book`、`/api/bookwithuser`、`/api/LendRecord`、`/api/user/usersearch`。
- Playwright 在两档视口实际点击“上架图书”和“新增读者”，对应弹窗均成功打开并关闭（`browser-check.json` 中 `dialogSmoke: true`）。
