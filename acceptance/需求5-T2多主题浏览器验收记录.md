# 需求 5 T2：多主题浏览器验收记录

日期：2026-07-17
范围：前端三主题全页面差异化布局、响应式、主题持久化与浏览器渲染。

## 实施结果

- Atlas：完整左栏、顶部状态栏、高密度筛选与数据表工作台。
- Academy：顶部横向导航、杂志式标题、左筛选/右数据纸张分栏；紧凑视口自动回落为纵向编排。
- Command：窄图标轨道、悬浮控制栏、深色网格内容、控制台式指标及数据面板。
- 所有主题共享原业务组件、路由和 methods，没有复制 API 请求或保存逻辑。
- PageHeader、BookForm、Element Plus 弹窗、表格、分页、下拉、消息、空状态均纳入主题边界。

## 自动化覆盖

浏览器：Playwright 1.61.1 / Chromium 149 Headless Shell。
地址：Docker frontend `http://localhost:9876`。

视口：

- 1440×900
- 1280×800
- 1024×768
- 768×1024

页面：

- Dashboard
- Book
- User
- BookWithUser
- LendRecord
- Person
- Password
- Register（三主题渲染检查）
- Login（三主题截图与真实管理员登录）

交互：

- 管理员 `admin` 登录成功。
- 三主题即时切换，URL 不变化。
- 刷新后主题与 localStorage 值一致。
- Book“上架图书”、User“新增读者”弹窗打开/取消通过。
- BookWithUser、LendRecord 的“仅看逾期未还”筛选与查询通过。
- Person 字段编辑/恢复、Password 重置、Register 角色切换与注册码字段显隐通过。
- 关键页面 API 响应均为 HTTP 200。

User 截图收口使用正式接口临时数据（本次运行）：

- 创建：`POST /api/user`，username=`theme_v2_28308887`，姓名=`多风格验收读者-28308887`，返回 HTTP 200、`code=0`。
- 创建后 `usersearch`：reader id=`156`，按姓名筛选 `total=1`，唯一记录为该中文读者。
- 在真实 User 页面按该姓名筛选后生成 `atlas-user.png`、`academy-user.png`、`command-user.png`，三张截图均只显示这一条中文记录。
- 清理：`DELETE /api/user/156?operatorId=<adminId>`，返回 HTTP 200、`code=0`。
- 清理后再次 `usersearch`：HTTP 200、`code=0`、`total=0`、`records=[]`。
- Playwright spec 将创建、筛选、截图、删除和零残留搜索置于可复现流程中，并通过 `finally` 清理。
