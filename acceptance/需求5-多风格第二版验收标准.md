# 需求 5 第二版：多风格与差异化布局验收标准

## 目标

在不改变现有业务接口、字段和操作规则的前提下，将前端升级为可即时切换的三套完整视觉与布局系统，并同步更新真实截图、演示脚本和最终汇报 PPT。

## 已确认产品决策

- 提供三套全站风格：
  1. **馆藏运营台**：深蓝＋青绿；完整左侧导航＋顶部状态栏；高信息密度。
  2. **书院阅读风**：米白＋砖红＋纸张语义；顶部横向导航；居中分栏与大留白。
  3. **数字指挥舱**：深色＋荧光蓝紫；窄图标侧栏＋悬浮工具栏；网格化数据区域。
- 首次访问默认使用“书院阅读风”。
- 登录页和登录后 Header 均提供风格选择器。
- 切换立即生效、不刷新页面，并通过 `localStorage` 持久化；不新增后端主题接口或数据库字段。
- 三套风格必须改变全部核心页面的布局，而非只换颜色。
- 覆盖登录、注册、Dashboard、图书、读者、当前借阅、借阅记录、个人信息、修改密码及公共 Header、导航、弹窗、表单、空状态。
- 业务按钮名称、字段、API 路径和业务流程保持一致。

## 可执行验收 Checklist

### A. 主题系统

- [x] 存在唯一的主题状态来源，三个稳定主题标识可切换。
- [x] 首次无保存值时默认“书院阅读风”。
- [x] 登录页和登录后页面均可切换三套风格。
- [x] 切换后当前页面立即变化，不发生整页刷新。
- [x] 刷新、退出登录、重新打开页面后保留上次选择。
- [x] 非法或旧的本地主题值安全回退到“书院阅读风”。

### B. 布局差异

- [x] 馆藏运营台使用完整左侧导航和顶部状态栏。
- [x] 书院阅读风使用顶部横向导航，主内容采用居中分栏/杂志式编排，不保留完整左栏占位。

## 禁止事项

- 不修改 `vue/src/utils/request.js`、`application.properties`、`Result.java` 或路由 history mode。
- 不修改后端业务接口数据结构，不新增仅用于主题的后端接口和数据库表。
- 不处理既有“普通读者可查看全部读者列表”权限 WARN。
- 不修改用户本地已有的 `验收.md`、`需求变化.md`、旧 PPT/HTML、`tools/` 和 `.pi/agents/library-worker.md`。
- 不用三份页面复制实现三套主题；主题差异应通过共享组件、状态、CSS tokens 和受控布局分支完成。
