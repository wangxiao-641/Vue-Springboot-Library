# 需求变化 1：支持图书库存数量——验收记录

验收日期：2026-07-16

## 改动范围

- 图书新增必须提交正整数 `totalCount`，后端初始化 `availableCount = totalCount`，拒绝客户端提交 `availableCount`。
- 管理员编辑馆藏总数时，后端按“当前已借出数 = 旧总数 - 旧可借数”重算新可借数量。
- 图书编辑与借还书使用同一 book 行锁串行化，禁止客户端修改系统维护的 `borrownum`；有当前借阅时禁止修改 ISBN 或删除图书。
- 借书/还书继续通过 `/circulation/*` 事务接口原子更新库存、当前借阅和历史记录。
- `bookwithuser` 增加 `borrow_id` 技术主键，使用 `(id, isbn)` 唯一约束，支持不同读者同时借同一 ISBN，并禁止同一读者重复借同一 ISBN。
- 图书页面显示馆藏总数和可借数量，新增/编辑只允许管理员填写馆藏总数。
- 新增 `verify_inventory_http.py` 纯 HTTP 黑盒验收脚本。

## 残余风险

- 旧 `/bookwithuser/*` 与 `/LendRecord/*` 直写接口仍可绕过 `CirculationService`；本需求的新页面流程和黑盒脚本使用事务接口，但旧接口属于项目已知遗留设计。
- 当前没有专用故障注入端点，HTTP 黑盒无法人为制造“库存已扣减后、后续 insert 抛数据库异常”的场景；事务方法使用 `@Transactional(rollbackFor = Exception.class)`，构建和正常/错误/并发路径均已验证。
- `verify-full` 继续报告既有权限警告：普通读者仍可查看所有读者列表，与库存需求无关。
