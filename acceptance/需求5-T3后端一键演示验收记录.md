## Review 修复：空管理员密码 fallback

- 修复前：显式 `ADMIN_PASSWORD=''` 仍会被 export，导致 Python 脚本认为调用方提供了空密码，不再尝试既有 fallback。
- 修复后：仅当 `${ADMIN_PASSWORD:-}` 非空时 export；空值或未设置时明确 `unset ADMIN_PASSWORD`。
- 无业务写入验证：临时 `/tmp` Python shim 代替四个黑盒入口，运行 `ADMIN_PASSWORD='' ./demo_backend_verification.sh`。四次调用均确认子进程中 `ADMIN_PASSWORD` 不存在，四项退出码均为 0，总退出码 0；shim 已删除。
- 真实回归：默认一键脚本 TOTAL PASS、退出码 0；显式 `ADMIN_PASSWORD=123456` 一键脚本 TOTAL PASS、退出码 0。
- `bash -n demo_backend_verification.sh`、`npm run verify:theme --prefix vue`、`npm run build --prefix vue` 均通过。
